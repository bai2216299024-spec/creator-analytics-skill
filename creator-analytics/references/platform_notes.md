# 创作者平台 — 平台说明与注意事项

## 小红书创作者平台 (creator.xiaohongshu.com)

### 页面结构
- **数据管理页**: `/publish/content` — 展示已发布的内容列表
- **单篇详情**: 点击内容标题可查看详细数据
- **内容列表**: 通常是以卡片或表格形式展示，包含缩略图、标题、发布时间、数据指标

### 已知选择器模式（可能随平台更新变化）
- 内容卡片: `.note-item`, `.post-item`, `[class*="note-list"] > div`
- 发布时间: `.publish-time`, `[class*="time"]`, 或卡片内的文本
- 播放量: `.play-count`, `[class*="play"]`, `[class*="view"]`
- 点赞量: `.like-count`, `[class*="like"]`
- 评论量: `.comment-count`, `[class*="comment"]`

### Cookie 说明
- 登录态 Cookie 通常有效期为 **7-30 天**
- 过期后访问会跳转到 `passport.xiaohongshu.com` 登录页
- 支持扫码登录和手机号验证码登录

### 常见问题
1. **页面改版导致选择器失效**: 采集器内置了多套备选选择器，但仍可能因大幅改版而失效。届时需要更新 `scrape_xhs.py` 中的选择器。
2. **反爬机制**: 小红书有较完善的反爬检测。使用 Playwright 全功能浏览器（非 headless 模式）可有效规避。
3. **数据加载延迟**: 数据管理页可能有懒加载，采集器会尝试滚动到底部触发加载。

## 抖音创作者平台 (creator.douyin.com)

### 页面结构
- **内容管理页**: `/creator-micro/content/manage` — 展示视频作品列表
- **列表形式**: 通常是卡片式布局，每张卡片包含封面、标题、数据指标
- **虚拟滚动**: 抖音使用虚拟滚动技术，需要持续滚动页面才能加载更多内容

### 已知选择器模式
- 内容卡片: `[class*="card"]`, `[class*="content-item"]`, `[class*="video-card"]`
- 标题: 卡片内的文本元素
- 播放量: `[class*="play"]`, 卡片中的数字文本
- 点赞量: `[class*="like"]`, 卡片中的数字文本
- 评论量: `[class*="comment"]`, 卡片中的数字文本

### Cookie 说明
- 抖音 Cookie 有效期通常为 **7-30 天**
- 过期后跳转到 `login.douyin.com` 或 `sso.douyin.com`
- 支持抖音 App 扫码登录

### 常见问题
1. **需要滚动多次加载**: 采集器会自动滚动 10 轮（每轮等待 3 秒），如果内容非常多可能需要增加轮数。
2. **虚拟 DOM 节点复用**: 抖音的虚拟滚动可能导致某些 DOM 节点不在当前可视区域，采集器专注于可见节点的文本提取。
3. **User-Agent 检测**: 采集器设置了 Chrome 120 的 User-Agent，如果遇到"请使用抖音 App 扫码"的提示，通常是 Cookie 过期。
4. **数据单位**: 抖音显示的数据可能缩写（如"1.2w"），采集器会自动解析。

## 通用注意事项

### 首次使用流程
1. 运行 `python3 scripts/run_all.py --headed`
2. 浏览器会打开小红书 → 扫码登录 → 关闭标签页
3. 浏览器打开抖音 → 扫码登录 → 关闭标签页
4. Cookies 保存后，后续运行无需再登录

### Cookie 管理
- Cookie 文件位置: `data/cookies/xhs_cookies.json` 和 `data/cookies/douyin_cookies.json`
- 删除这两个文件可以强制重新登录
- 每个文件包含完整的 Playwright storage state（cookies + localStorage）

### 采集策略
- 使用 Playwright 全功能浏览器，模拟真实用户操作
- 自动滚动加载更多内容
- 多套备选选择器，提高容错性
- 文本级备选方案，即使特定选择器失效也能提取数据

### 已知限制
1. **无法采集付费/合作内容**: 只有公开可见的内容会出现在管理列表中
2. **数据并非实时**: 平台的数据可能有数小时到一天的延迟
3. **只能采集已发布内容**: 草稿和审核中的内容不在数据管理列表中
4. **单次采集上限**: 小红书和抖音的管理列表通常只显示最近 30-90 天的内容

## 排错指南

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 浏览器打开后空白 | 网络问题或平台改版 | 检查网络，确认 URL 是否正确 |
| 跳转到登录页 | Cookie 过期 | 重新运行 `--headed` 扫码登录 |
| 找不到内容列表 | 选择器失效 | 检查页面元素，更新 `scrape_xhs.py` / `scrape_douyin.py` 中的选择器 |
| 数字指标全是 0 | 解析逻辑问题 | 检查页面实际显示的文本格式，更新 `parse_number()` |
| 提示"请使用 App 扫码" | 反爬检测 | 使用 `--headed` 模式或更新 User-Agent |
| Playwright 报错 | 依赖缺失 | 重新运行 `scripts/setup.sh` |
